# 기능 + UX 동시 합격 게이트 정책 (초보자용)

## 왜 이 문서가 필요한가?
이 정책은 **"기능은 되는데 쓰기 불편한 상태"**를 막기 위한 기준입니다.

- 기능 체크리스트: 앱이 기술적으로 제대로 동작하는지 확인
- UX 체크리스트: 사람이 실제로 빠르고 실수 없이 쓸 수 있는지 확인

즉, 앞으로 변경(개발/수정)마다 **기능 + UX를 동시에 통과**해야만 머지(반영)할 수 있습니다.

---

## 1) Fixture 5종 기준 기능 체크리스트

> Fixture = 테스트용 예제 HTML 파일입니다.

### Fixture 목록
- F01: `fixture_01_blank_builder_860.html`
- F02: `fixture_02_sample_shop_builder_860.html`
- F03: `fixture_03_sample_template_existing_html.html`
- F04: `fixture_04_sample_dring_walk_allinone.html`
- F05: `fixture_05_user_melting_cheese_compact.html`

### 공통 기능 체크 항목(모든 Fixture 공통)

| 체크 항목 | 확인 방법 | 합격 기준 |
|---|---|---|
| 슬롯 감지 | 파일 열기 후 슬롯 개수/선택 가능 여부 확인 | 각 fixture의 기대 슬롯이 감지되고 편집 가능 |
| 저장(Linked) | linked HTML 저장 후 재오픈 | 이미지/텍스트/레이아웃 변경이 유지 |
| 저장(Embedded) | embedded HTML 저장 후 재오픈 | 외부 파일 없이 동일 결과 유지 |
| Export(전체 PNG) | 전체 PNG 내보내기 실행 | 잘림/누락 없이 생성 |
| Export(분할 PNG) | 분할 PNG 내보내기 실행 | 섹션 순서/개수 정상 |
| 경로 보존 | 상대경로 + `uploaded:` + 기존 이미지 확인 | 재오픈/export 후 깨짐 없음 |

### Fixture별 최소 확인 포인트

| Fixture | 슬롯 | 저장 | Export | 경로 보존 |
|---|---|---|---|---|
| F01 | 기본 슬롯 감지/편집 가능 | linked/embedded 둘 다 재오픈 성공 | 전체/분할 PNG 성공 | 상대경로 깨짐 없음 |
| F02 | 다중 드롭 순서 정상 | 저장 후 순서/배치 유지 | 분할 순서 일치 | 상대경로 자산 유지 |
| F03 | 기존 템플릿 슬롯 오탐 최소 | 기존 구조/스타일 유지 저장 | 전체 PNG 시 레이아웃 유지 | 기존 이미지 경로 유지 |
| F04 | heuristic 슬롯 다수 감지 | 경고 카드 포함 저장 일관성 | 분할 PNG 섹션 정합성 | 혼합 경로(상대/업로드) 유지 |
| F05 | 기존 이미지 포함 슬롯 보존 | 재오픈 시 13개 슬롯 회귀 없음 | 전체/분할 모두 누락 없음 | `uploaded:` 스킴 포함 보존 |

## 2) UX 체크리스트(시나리오 단위)

> UX 평가는 "한 명의 신규 사용자" 기준으로 측정합니다.

### 공통 측정 지표 정의
- 작업 완료 시간: 시작~완료까지 걸린 시간(초)
- 클릭 수: 마우스 클릭/탭 횟수
- 오류율: 실패 행동 비율 (`실패 행동 수 / 전체 행동 수`)

### 시나리오별 UX 최소 기준

| 시나리오 ID | 작업 완료 시간 | 클릭 수 | 오류율 |
|---|---:|---:|---:|
| S1 이미지 교체 중심 | 90초 이하 | 14회 이하 | 5% 이하 |
| S2 텍스트+레이아웃 수정 | 180초 이하 | 28회 이하 | 8% 이하 |
| S3 최종 export | 120초 이하 | 16회 이하 | 5% 이하 |

> 팁: 초보자 온보딩 직후(설명 1회) 기준으로 측정하고, 3회 반복 평균값으로 합격/불합격을 판단합니다.

---

## 3) 대표 시나리오 3개 고정

### S1. 이미지 교체 중심 (고정)
1. fixture 열기 (권장: F05)
2. 기존 이미지 슬롯 2개 중 1개 교체
3. 빈 슬롯 2개에 추가 업로드
4. cover/contain 1회 변경
5. linked 저장 후 재오픈 검증

### S2. 텍스트 + 레이아웃 수정 (고정)
1. fixture 열기 (권장: F03 또는 F04)
2. 제목/본문 텍스트 수정
3. 텍스트 박스 크기/위치 조정
4. 인접 요소와 겹침 없는지 확인
5. embedded 저장 후 재오픈 검증

### S3. 최종 export (고정)
1. fixture 열기 후 최종 상태 준비
2. 전체 PNG export
3. 분할 PNG export
4. 파일 개수/순서/잘림 여부 확인

## 4) 머지 게이트 규칙 (기능 + UX 동시 통과)

다음 규칙을 모두 만족해야 머지 가능합니다.

1. 기능 체크리스트: Fixture 5종 모두 PASS
2. UX 체크리스트: S1/S2/S3 모두 최소 기준 충족
3. 둘 중 하나라도 FAIL이면 머지 금지

### 판정 표준
- **PASS**: 기능/UX 모두 통과
- **HOLD**: 기능 통과 + UX 미달 (머지 보류)
- **FAIL**: 기능 실패 (즉시 수정 필요)

---

## 5) 실패 항목 분류 규칙

기존처럼 전부 "버그"로 묶지 않고 아래처럼 분리 추적합니다.

- 기능 실패: `bug` 라벨
- 기능은 통과했지만 사용감이 나쁨: **`usability-defect` 라벨**

### `usability-defect` 라벨 예시
- 작업 완료 시간이 기준보다 지속적으로 느림
- 클릭 수가 과도하게 많음(불필요 단계 존재)
- 실수 유도 UI(버튼 의미 불명확, 되돌리기 어려움)
- 저장/export는 되지만 사용자가 반복적으로 경로를 헷갈림

---

## 운영 요약 (한 줄 버전)

앞으로 모든 변경은 **"기능 통과 + UX 최소 기준 통과"**를 동시에 만족해야 머지하며,
UX 미달 항목은 버그가 아닌 **`usability-defect`**로 별도 관리합니다.
