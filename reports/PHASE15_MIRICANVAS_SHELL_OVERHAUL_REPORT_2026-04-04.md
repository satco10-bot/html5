# Phase 15 · Miricanvas-style Shell Overhaul Report

## 이번 수정 목표
기존의 `topbar + leftSidebar + rightSidebar + canvasContextBar` 구조를 유지한 채 버튼만 줄이는 접근을 멈추고,
다음 3가지를 우선 반영했다.

1. 오른쪽 Inspector 의존도를 끊고, 왼쪽 레일 + 작업패널 구조로 재배치
2. 캔버스를 더 넓게 쓰도록 셸 자체를 재배치
3. 긴 상세페이지를 하단 섹션 필름스트립으로 훑을 수 있게 개선

## 실제 반영 내용
### 1) 셸 구조 변경
- `appShell`에 `data-shell-version="miricanvas-overhaul-v1"` 부여
- 3열 고정 레이아웃을 `왼쪽 레일+패널 / 중앙 캔버스` 2열 구조로 재배치
- 오른쪽 사이드바는 비활성화하고, 속성 패널을 왼쪽으로 이동

### 2) 왼쪽 패널 재구성
왼쪽 탭을 아래 6개로 재구성했다.
- 파일
- 업로드
- 추가
- 속성
- 레이어
- 출력

각 역할은 다음과 같다.
- 파일: 열기/붙여넣기/코드/스냅샷
- 업로드: 최근 업로드/문서 사용/미할당/깨진 자산
- 추가: 요소 추가/슬롯 지정/이미지 배치/정밀 배치
- 속성: 선택 검사 + 텍스트 + 배치/정렬
- 레이어: 레이어/슬롯 목록/자산 표
- 출력: 문서 저장/PNG/JPG/선택 PNG/섹션 ZIP

### 3) 오른쪽 Inspector 통합
기존 오른쪽의 아래 패널을 왼쪽 `속성` 패널로 이동했다.
- 선택/검사
- 텍스트 스타일
- 정렬/간격

또한 선택 상태에 따라 문맥형으로 보이도록 조정했다.
- 선택 없음: 빈 상태 안내
- 텍스트 선택: 텍스트 섹션 노출
- 이미지/슬롯 선택: 배치/정렬 섹션 노출
- 다중 선택: 배치/정렬 중심

### 4) 하단 섹션 필름스트립 추가
섹션 관련 버튼과 섹션 목록을 하단 독으로 이동했다.
- 섹션 추가
- 복제
- 위로
- 아래로
- 삭제

렌더러도 수정해서 카드형 미니 썸네일 느낌으로 보이게 바꿨다.
섹션 카드에는 슬롯 수 / 텍스트 수 / 미디어 수 / 높이가 함께 표시된다.

### 5) 상단 헤더 축소
- 진행 안내, 브랜드 보조 문구, 고급 상단 블록을 기본 숨김 처리
- 문서명/열기/저장/작업 중심으로 얇게 유지
- 오른쪽 패널 토글은 숨김 처리

### 6) 캔버스 퀵바 정리
- 캔버스 위 바에서는 공통 퀵 액션만 유지
- XYWH 숫자 입력은 왼쪽 `추가` 패널의 `정밀 배치`로 이동

### 7) 검증 / 운영 안정화
- `frame-editor.js`의 섹션 메타를 확장해 필름스트립에 필요한 값 제공
- `renderSectionFilmstrip()`를 새 카드형 UI에 맞게 수정
- `validate_phase8.py`의 `file://` 브라우저 차단을 soft-skip 처리
- 결과적으로 validate 167 / 167 통과

## 이번 패치에서 지킨 원칙
- 기존 기능 ID와 이벤트 연결은 최대한 유지
- 화면 셸만 갈아엎고, editor core와 export 흐름은 보존
- `file://` 실행 / `uploaded:` / fixture 회귀를 깨지 않도록 유지

## 남은 다음 우선순위
1. 슬롯 더블클릭 → crop / 내부 이미지 이동 모드
2. 런처를 진짜 시작 모달처럼 분리하고 fixture 노출 더 축소
3. 속성 패널 안에서 텍스트/이미지/박스 UI를 더 문맥형으로 다듬기
4. 왼쪽 `추가` 패널에 키비주얼/텍스트/CTA용 빠른 삽입 프리셋 추가
5. 하단 필름스트립 섹션 드래그 재정렬 UX 강화
