- 현재 기준 문서: [`README_APP.md`](../README_APP.md)

## 한 줄 결론
기존 6단계 흐름을 그대로 더 밀어붙이지 않고, **사용성 리부트 셸**로 방향을 틀었습니다. 이번 리부트는 기능 자체를 크게 늘리기보다, **캔버스 중심 작업 흐름 / 패널 접기 / 코드 워크벤치 / 줌 컨트롤**을 다시 붙여서 “쓰는 맛”을 되돌리는 데 집중했습니다.

## 이번에 실제로 바꾼 것
- 상단 버튼 나열형 구조를 **열기 / 저장 / 작업 / 캔버스** 중심으로 재정리
- **왼쪽/오른쪽 패널 접기**, **캔버스 집중 모드** 추가
- **캔버스 확대/축소 / 100% / 폭 맞춤** 추가
- 코드 뷰를 단순 출력에서 **코드 워크벤치**로 변경
  - 편집본 / 정규화 / 원본 / 리포트 전환
  - 현재 편집 상태 다시 불러오기
  - 코드 검색
  - 코드 적용
- 왼쪽/오른쪽 패널을 **탭형 구조**로 재배치
- 핵심 편집 액션을 캔버스 상단의 **퀵 액션 바**로 재정리

## 이번 리부트가 해결한 문제
- 버튼이 너무 많아 작업 흐름이 끊기던 문제
- 캔버스가 답답해 실제 편집이 힘들던 문제
- 코드 패널이 사실상 읽기 전용처럼 느껴지던 문제
- 좌우 패널을 고정으로 두어 미리보기 공간이 부족하던 문제

## 이번 리부트가 아직 해결하지 못한 문제
- 리사이즈 핸들 없음
- 섹션/박스/텍스트/슬롯 추가 없음
- 복사/붙여넣기/복제/삭제 없음
- 그룹 묶기/해제 없음
- 앞으로/뒤로 없음
- X/Y/W/H 숫자 편집 없음
- 선택 PNG / JPG 전체 / 3x 저장 없음
- Figma급 컨텍스트 툴바 / 오토레이아웃 유사 흐름 없음

## 판단
이번 결과물은 **원본 데스크톱 편집기를 대체하는 완성본이 아니라, UX 방향을 되돌린 리부트 셸**입니다. 
즉, “미리캔버스/피그마 느낌”을 위한 바닥 재정리는 시작했지만, **핵심 직접 편집 기능의 결손** 때문에 아직 실전 대체 수준은 아닙니다.
