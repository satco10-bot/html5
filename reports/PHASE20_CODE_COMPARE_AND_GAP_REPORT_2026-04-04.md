# PHASE20 코드 비교형 워크벤치 + 갭/버그 보고서

## 이번에 실제로 반영한 것

### 1) 코드 워크벤치에 “진짜 비교 보기” 추가
- `기준 / 비교` 소스를 각각 선택할 수 있게 했습니다.
- 비교 가능한 소스:
  - 현재 초안
  - 현재 보기
  - 편집본 HTML
  - 정규화 HTML
  - 원본 HTML
- 요약 한 줄이 아니라 **변경 덩어리(chunk)** 단위로 차이를 보게 만들었습니다.
- 각 변경 덩어리마다
  - 수정 / 추가 / 삭제 구분
  - 기준 줄 범위
  - 비교 줄 범위
  - 양쪽 코드 조각
  - 해당 줄로 점프 버튼
  을 제공합니다.
- 즉 이제는 “몇 줄 바뀌었는지”만 보는 게 아니라, **어디가 어떻게 달라졌는지** 빠르게 훑을 수 있습니다.

### 2) HTML-first 편집 흐름 강화
- 이 앱은 어디까지나 **HTML 기반 편집기**라서,
  - 코드 패널에서 수정
  - 캔버스에서 수정
  - 다시 코드로 비교/검토
  이 왕복이 핵심입니다.
- 이번 비교 패널은 이 HTML 왕복 흐름을 위한 기능입니다.

### 3) 검증 보강
- `validate_phase8.py`에 비교 패널 핵심 ID 존재 확인을 추가했습니다.
- 결과: 전체 검증 **172 / 172 통과**

## 지금 여전히 남아 있는 주요 버그/거친 부분

### A. 편집기 복잡도 자체는 아직 높습니다
- 기능은 강해졌지만 UI 요소 수가 여전히 많습니다.
- 특히 상단/왼쪽/캔버스 퀵바에 기능이 분산되어 있어, 완전히 “손에 붙는” 단계는 아닙니다.

### B. 코드 비교는 아직 “실전 diff 엔진”까지는 아닙니다
- 지금은 **라인 단위 chunk 미리보기**입니다.
- 길게 꼬인 대규모 재배열 HTML에서는 git diff 수준으로 정교하지 않을 수 있습니다.
- 하지만 로컬 HTML 편집기 문맥에서는 가볍고 빠르게 보는 용도로 충분한 1차 완성입니다.

### C. 브라우저 자동 스모크는 환경 차단 이슈가 남아 있습니다
- `file://` 페이지에 대한 headless 접근은 현재 환경에서 blocked입니다.
- 그래서 실제 브라우저 자동 클릭 스모크는 soft-skip입니다.
- 이건 앱 버그라기보다 실행 환경 제약입니다.

### D. 섹션/크롭/선택은 좋아졌지만 “미세 디테일”은 더 갈 수 있습니다
- 섹션 드래그 재정렬은 됨
- 슬롯 더블클릭 크롭도 됨
- 하지만 더 세밀한 HUD, 더 짧은 액션 흐름, 더 적은 버튼 수는 아직 개선 여지가 큽니다.

## 미리캔버스와 아직 다른 점

### 1) 정보 구조가 아직 완전히 미리캔버스식으로 정리되진 않았습니다
- 미리캔버스식 좋은 점은 “추가 / 편집 / 레이어 / 출력” 흐름의 질서인데,
  현재 앱은 아직 기능이 비교적 많이 노출된 편입니다.

### 2) 도구 레일의 밀도 조절이 더 필요합니다
- 지금도 많이 정리됐지만,
  여전히 초보 사용자 기준으로는 선택지가 많습니다.
- 미리캔버스 쪽은 “자주 쓰는 것만 먼저 보이고 나머지는 늦게 등장”하는 느낌이 더 강합니다.

### 3) 선택 후 플로팅 퀵바가 더 짧고 날렵해질 수 있습니다
- 현재도 문맥형이지만,
  더 과감하게 4~6개 핵심 액션만 남기고 나머지는 `…`로 보내는 쪽이 더 가깝습니다.

### 4) 슬롯/이미지 크롭 HUD는 더 시각적일 수 있습니다
- 지금은 기능적으로는 충분히 쓸만하지만,
  미리캔버스급으로 가려면
  - 배율 HUD
  - 적용/취소 chip
  - grid/guide 가시성
  - 잠금 상태 표현
  을 더 정교하게 다듬을 수 있습니다.

### 5) 섹션 네비게이터는 아직 “필름스트립 최종형”이 아닙니다
- 현재는 실제 미리보기형 카드에 가까워졌지만,
  더 작은 썸네일 + 이름 + 드래그 + 점프 + 메뉴가 더 압축적으로 붙어야 합니다.

## 다음 개선사항 준비 목록 (우선순위)

### 1순위 — 사용성 체감 큰 것
1. **코드 diff 2단계**
   - 기준/비교 고정 프리셋 버튼
   - `편집본 ↔ 원본`, `편집본 ↔ 정규화`, `초안 ↔ 현재 보기` 빠른 전환
   - diff chunk 접기/펼치기
2. **플로팅 퀵바 압축**
   - 1차 액션 4~6개만 노출
   - 나머지는 `…` 메뉴로 이동
3. **좌측 레일 단계화**
   - 자주 쓰는 탭 우선
   - 고급은 접기

### 2순위 — 편집 손맛 강화
4. **크롭 HUD 2차**
   - 배율 표시 더 명확히
   - 드래그 중 좌표/오프셋 힌트
   - 적용/취소 위치 단순화
5. **선택 영역 안정화 2차**
   - 작은 요소/겹친 요소에서 스마트 선택 예측성 강화
   - 마퀴 선택 bbox 피드백 개선
6. **섹션 카드 고도화**
   - 드래그 핸들 명확화
   - 섹션 카드 `···` 메뉴 정교화
   - 다중 섹션 작업 준비

### 3순위 — HTML 기반 앱다운 강화
7. **코드 diff + 적용 전 검사 연동**
   - diff에서 위험 chunk 강조
   - 검사 결과를 diff chunk와 연결
8. **코드 오류 복구 UX**
   - 잘못된 HTML 적용 시 마지막 정상 상태 복귀 옵션
9. **코드/캔버스 동기화 히스토리**
   - “캔버스에서 마지막으로 코드로 보낸 시점” 기록
   - “코드에서 마지막으로 캔버스로 적용한 시점” 기록
