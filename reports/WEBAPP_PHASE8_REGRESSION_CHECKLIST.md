- 기준 커밋: `60b1f586154070c5a2bd456cffa011e1e8ef2b80`
- 기준 날짜: `2026-04-03`
- 현재 기준 문서: [`README_APP.md`](../README_APP.md)

## 목적
- 레이어 패널에서 한 조작 결과가 캔버스에 즉시 반영되는지 확인한다.
- 캔버스에서 한 조작 결과가 레이어 패널에 즉시 반영되는지 확인한다.

## 테스트 항목

1. **캔버스 선택 → 레이어 패널 하이라이트 동기화**
   - 캔버스에서 요소 1개 선택.
   - 기대 결과: 동일 `data-node-uid` 레이어 행이 `is-active` 상태로 강조된다.

2. **레이어 패널 선택 → 캔버스 선택 테두리 동기화**
   - 레이어 패널의 임의 요소 클릭.
   - 기대 결과: 캔버스 해당 요소가 선택 상태 테두리로 표시되고 선택 인스펙터가 갱신된다.

3. **다중 선택 동기화**
   - 캔버스 Shift+드래그(또는 Ctrl/Cmd 클릭)로 2개 이상 선택.
   - 기대 결과: 레이어 패널에서도 동일 uid 집합이 동시에 active 표시된다.

4. **삭제 동기화**
   - 다중 선택 후 삭제.
   - 기대 결과: 캔버스에서 요소가 제거되고 레이어 패널 목록에서도 즉시 사라진다.

5. **Undo 복원 동기화**
   - 삭제 직후 Undo 실행.
   - 기대 결과: 삭제된 요소가 캔버스/레이어 패널 모두 같은 순서로 복원된다.

6. **z-order 동기화 (앞으로/뒤로/맨앞/맨뒤)**
   - 선택 요소에 대해 4개 버튼을 순서대로 수행.
   - 기대 결과: 캔버스의 겹침 순서와 레이어 패널의 목록 순서가 항상 일치한다.

7. **복제 동기화**
   - 복제 실행.
   - 기대 결과: 복제 요소가 (+10,+10) 위치 오프셋으로 생성되고 레이어 패널에 즉시 추가된다.

8. **새 요소 추가 동기화 (텍스트/박스/이미지 슬롯)**
   - 각 추가 버튼 1회씩 실행.
   - 기대 결과: 캔버스에 기본 프리셋 스타일로 생성되고 레이어 패널에도 즉시 표시된다.

## 자동 점검 스크립트
- `python3 scripts/regression_layer_canvas_sync.py`
