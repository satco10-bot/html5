# 웹앱 갭 감사 + 리부트 계획

## 현재 상태를 냉정하게 보면
지금 웹앱은 “6단계까지 갔다”는 표현과 달리, **실전 대체 도구로는 아직 부족합니다.**
문제는 단순히 기능 수가 모자라서가 아니라, **직접 편집의 핵심 동선**이 비어 있기 때문입니다.

### 제가 판단한 현재 수치
- **원본 데스크톱 기능 패리티:** 약 **52%**
- **체크박스형 전체 기능 커버리지:** 약 **55%**
- **실전 대체 가능성:** 약 **35%**
- **미리캔버스/피그마급 UX 달성률:** 약 **15% 내외**

위 숫자는 마케팅용 수치가 아니라, 실제 코드와 UI를 기준으로 **불러오기 / 슬롯 / 직접조작 / 저장 / UX**를 나눠서 본 체감치입니다.

## 왜 지금 사용성이 더 나빠졌는가

### 1. 직접 편집 핵심이 빠져 있습니다
원본 데스크톱은 적어도 아래가 있었습니다.
- 리사이즈 핸들
- 섹션/박스/텍스트/슬롯 추가
- 복사/붙여넣기/복제/삭제
- 그룹 묶기/해제
- 앞으로/뒤로
- X/Y/W/H 숫자 편집
- 이미지 확대/축소 / nudge

현재 웹앱은 슬롯 감지, 이미지 넣기, 텍스트 수정, 드래그 이동은 되지만,
**“구조를 고치는 편집”**이 빠져 있어서 실전 편집 밀도가 확 떨어집니다.

### 2. 인터페이스가 캔버스보다 패널 중심이었습니다
이전 단계는 상단 버튼이 너무 많고, 패널이 계속 시야를 차지했습니다.
그래서 사용자가 느끼기엔 “무엇이 가능한지는 많아 보이는데, 실제로 손이 가는 편집은 어렵다”가 됩니다.

### 3. 기능 추가 순서가 잘못됐습니다
출력 전 검수, 자산 레지스트리, 레이어 숨김/잠금은 분명 가치가 있지만,
그 전에 **직접 조작 도구**가 먼저 완성됐어야 했습니다.

## 그래서 계획을 바꿔야 합니다
기존 7단계, 8단계로 그냥 계속 가면 안 됩니다.
이제는 **Phase 7, 8**이 아니라 아래처럼 **리부트 스프린트**로 전환하는 게 맞습니다.

## 리부트 스프린트 제안

### R1. 데스크톱 패리티 회복
가장 먼저 이걸 채워야 합니다.
- 리사이즈 핸들
- 복사 / 붙여넣기 / 복제 / 삭제
- 섹션 / 박스 / 텍스트 / 슬롯 추가
- 앞으로 / 뒤로
- X / Y / W / H 숫자 편집
- 선택 영역 PNG
- JPG 전체
- 3x 저장
- 이미지 확대 / 축소 / nudge

이 단계가 끝나야 “예전 것보다 못하다”는 말이 줄어듭니다.

### R2. 캔버스 UX 복원
- 왼쪽 툴 레일
- hand/pan, zoom slider
- 선택 박스와 resize handle 시각 강화
- 컨텍스트 퀵바
- 패널 접기/펼치기 기억
- 단축키 밀도 복원

### R3. 구조 편집 고도화
- 그룹 묶기/해제
- 프레임 개념
- 오토레이아웃 비슷한 정렬 보조
- 반복 카드/옵션 블록 템플릿
- 슬롯 라벨/태그 기반 일괄 치환

### R4. 출력/검수 정리
- linked HTML / embedded HTML 둘 다 복원
- 내장→링크 전환
- export preset 정교화
- 깨진 자산/빈 슬롯/원격 폰트 경고
- 저장 형식 안정화

## 다음 구현 우선순위
지금 바로 다음으로 해야 할 건 **R1**입니다.
특히 아래 6개를 먼저 붙여야 체감이 확 달라집니다.
1. 리사이즈 핸들
2. 복제/삭제
3. 박스/텍스트/슬롯 추가
4. X/Y/W/H 패널
5. 앞으로/뒤로
6. JPG / 선택 PNG / 3x 저장

## 이번 리부트 셸에서 시작한 보정
이번에는 전체 구조를 다시 보면서 먼저 아래를 반영했습니다.
- 패널 접기 / 캔버스 집중 모드
- 캔버스 확대/축소/폭맞춤
- 코드 워크벤치 복원
- 탭형 패널 구조
- 퀵 액션 바 재정리

이건 방향 수정용 첫걸음이고, **핵심 직접 편집 기능 회복이 본게임**입니다.
