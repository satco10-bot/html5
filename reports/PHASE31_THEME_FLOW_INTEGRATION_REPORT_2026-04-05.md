# PHASE 31 — 테마/코드/검사/적용 흐름 통합 보고서

## 이번 라운드 목표
흩어져 있던 테마/토큰/검사/코드 적용 기능을 한 흐름으로 정리해서, HTML 기반 상세페이지 편집기다운 작업 동선을 만들었습니다.

## 실제 반영 내용
- 좌측 사이드바에 `테마` 탭 추가
- `색상 / 토큰 / 검사 / 적용` 4모드로 테마 스튜디오 구성
- 기존 기능 노드를 런타임에서 새 테마 패널로 이동
  - 색상: 스타일 컬러 스튜디오, 섹션별 테마 팔레트
  - 토큰: CSS 컬러 토큰, typography/spacing/radius/shadow 토큰, inline 색상 추출, 슬롯 schema
  - 검사: 깨진 자산 relink, 마켓 lint, 대비 lint
  - 적용: 캔버스→HTML, HTML→캔버스, 안전 적용, 빠른 비교, 이슈/색상 필터
- 코드 워크벤치와 테마 스튜디오를 연결하는 상태 요약/버튼 추가
- 워크플로 매핑에 `left-theme`를 편집 단계로 반영
- 마켓 lint 결과 캐시 / CSS 토큰 캐시를 테마 스튜디오 요약에 연결

## 구현 메모
- 기존 ID/이벤트를 보존하기 위해 기능 섹션을 새 패널로 **이동(mount)** 하는 방식으로 정리했습니다.
- 덕분에 기존 이벤트 핸들러를 대부분 유지하면서도 사용 흐름은 한곳에 모을 수 있었습니다.
- HTML ↔ 캔버스 왕복과 검사/안전 적용 흐름을 한 패널에서 이어서 처리할 수 있습니다.

## 현실 평가
좋아진 점:
- 색상/토큰/검사/적용 흐름이 분산되지 않고 한쪽으로 모였습니다.
- HTML 편집기 특화 기능이 훨씬 찾기 쉬워졌습니다.
- 코드 패널을 완전히 열지 않아도 안전 적용/검사/빠른 비교까지 접근 가능합니다.

남은 점:
- 정적 HTML 기준 버튼 수는 오히려 늘었습니다. 이번 라운드는 **노출 감소보다 흐름 통합**이 우선이었습니다.
- 테마 패널 안의 각 세부 섹션도 다음에는 한 단계 더 압축할 수 있습니다.
- 런타임 이동 방식이라, 다음 라운드에는 아예 HTML 구조를 정리해 정적 DOM도 간소화하는 편이 낫습니다.
