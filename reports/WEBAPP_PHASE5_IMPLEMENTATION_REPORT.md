# 상세페이지 웹앱 5단계 구현 보고서 (로컬 전용)

## 이번 단계 목적

4단계까지 만든 로컬 전용 웹앱 위에, 실제 상세페이지 실무에서 자주 필요한 다음 4가지를 붙였습니다.

1. 레이어 패널
2. 멀티선택 + 정렬/간격 작업
3. 고급 텍스트 스타일 패널
4. 출력 전 검수(preflight)

이번 단계도 **서버/도메인 없이 `file://`로 여는 개인용 로컬 웹앱** 기준을 유지했습니다.

---

## 로컬 전용 기준에서 유지한 것

- `index.html` 직접 실행
- `type="module"` 제거 유지
- 단일 `app.bundle.js` 유지
- 시작 시 `fetch()` 의존 없음
- File System Access API 미사용
- `<input type="file">` + 브라우저 다운로드 + `localStorage` autosave 흐름 유지
- 미리보기는 `iframe.srcdoc` 유지

즉 이번 단계 기능 추가는 모두 **로컬 브라우저 안에서 끝나는 편집 경험** 위에 얹었습니다.

---

## 실제로 구현한 기능

### 1) 레이어 패널

오른쪽 사이드바에 레이어 패널을 추가했습니다.

- section / slot / text / box를 계층형으로 표시
- 레이어 검색 지원
- 레이어 클릭으로 바로 선택
- Ctrl/Cmd 클릭으로 다중 선택
- 현재 선택 상태를 레이어 패널과 선택 진단에 동기화

기존 슬롯 목록은 그대로 유지했고, 레이어 패널은 더 넓은 범위의 편집 대상을 잡는 용도로 분리했습니다.

### 2) 멀티선택 + 정렬/간격

미리보기 클릭과 레이어 패널 클릭 모두에서 다중 선택을 지원하도록 바꿨습니다.

지원 액션:

- 같은 너비
- 같은 높이
- 같은 크기
- 왼쪽 정렬
- 가운데 정렬
- 오른쪽 정렬
- 위쪽 정렬
- 세로 중앙 정렬
- 아래쪽 정렬
- 가로 분배
- 세로 분배
- 배치 이동 초기화

현재 단계의 정렬/분배는 **transform: translate() 기반**입니다.
장점은 기존 HTML flow를 완전히 깨뜨리지 않고 로컬 미리보기와 export에 바로 반영된다는 점입니다.
반면 CSS flow 자체를 재계산하는 스마트 레이아웃 엔진은 아직 아닙니다.

### 3) 텍스트 스타일 패널

텍스트 선택 시 우측 패널에서 바로 스타일을 조절할 수 있게 넣었습니다.

지원 항목:

- 글자 크기
- 줄간격
- 자간
- 굵기
- 글자색
- 왼쪽 / 가운데 / 오른쪽 정렬
- 인라인 텍스트 스타일 비우기

기존 더블클릭 직접 편집은 그대로 유지하면서,
이번 단계에서는 **“내용 편집”과 “스타일 편집”을 분리**해서 사용할 수 있게 했습니다.

### 4) 출력 전 검수(preflight)

오른쪽 패널에 출력 전 검수 결과를 추가했습니다.

검수 항목:

- 빈 슬롯
- 미해결 자산
- 원격 stylesheet 포함 여부
- 텍스트 편집 중 여부
- near miss 슬롯 후보
- fixture 기준 슬롯 수 차이

또한 PNG/ZIP export 직전에 **blocking error가 있으면 경고 confirm**을 띄우도록 넣었습니다.
즉, 사용자가 빈 슬롯이 남은 상태에서 그대로 PNG를 뽑는 사고를 줄이려는 목적입니다.

## 이번 단계에서 같이 고친 부분

### 스타일 저장 누락 가능성 보강

기존 구조에서는 원래 inline style이 있던 요소를 편집한 뒤,
저장 시 `data-original-style`로 되돌아가면서 일부 변형이 사라질 여지가 있었습니다.

그래서 이번 단계에서는 **편집 중 `setInlineStyle()`로 바뀐 스타일을 `data-export-style`로 항상 갱신**하도록 바꿨습니다.
이 조정으로 아래 작업이 저장/export 시 더 안정적으로 유지됩니다.

- 텍스트 스타일 수정
- 정렬/분배용 transform 이동
- same width / same height / same size
- 이미지 프리셋에서 생긴 style 변화

### 이미지 원복 시 stale export style 제거

슬롯 이미지를 원상복구할 때 예전 `data-export-style`이 남아 있으면,
저장 단계에서 복구 전 스타일이 다시 살아날 여지가 있었습니다.

이 부분은 슬롯 복구 시 `data-export-style`을 같이 제거하도록 정리했습니다.

## 현재 남아 있는 한계

1. 정렬/분배는 아직 flow 재배치가 아니라 translate 기반입니다.
2. 레이어 패널은 접기/펼치기와 drag reorder까지는 아직 아닙니다.
3. 멀티선택 상태 전체를 history snapshot으로 완전 복원하는 수준까지는 아직 아닙니다.
4. export는 여전히 브라우저의 foreignObject 렌더링에 의존하므로, 외부 폰트/CSS가 섞인 문서는 브라우저별 차이가 남을 수 있습니다.
