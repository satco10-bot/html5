# 시각 회귀 체크리스트 (Launcher / 편집 / 내보내기)

> 목적: `styles.css` UI 토큰/버튼/패널-모달-카드 스타일 통일 이후, 눈으로 빠르게 이상 여부를 확인하기 위한 체크리스트.

## 1) 런처(상단 Topbar) 화면
- [ ] 기본 버튼(`.button`)이 파란색 Primary로 보이고 텍스트 대비가 충분하다.
- [ ] 보조 버튼(`.button--secondary`)이 연한 파랑 배경 + 파란 텍스트로 일관된다.
- [ ] 고스트 버튼(`.button--ghost`)이 흰 배경 + 테두리로 보인다.
- [ ] 경고 버튼(`.button--warn`)이 연한 주황 계열로 표시된다.
- [ ] 그룹 카드(Topbar block)의 테두리/둥근 모서리/그림자 강도가 과하지 않고 읽기 편하다.

## 2) 편집(캔버스 + 사이드 패널) 화면
- [ ] 좌/우 패널과 중앙 스테이지가 동일한 표면 규칙(테두리/그림자)을 사용한다.
- [ ] 카드성 컴포넌트(메트릭 카드, 슬롯 카드) 간 시각 톤이 맞고 과도한 그림자가 없다.
- [ ] 작은 버튼(`.button--small`)의 높이/글자 크기가 균일하다.
- [ ] 선택/활성 상태와 일반 상태가 충분히 구분되지만 색상 과포화는 없다.
- [ ] 텍스트 크기(제목/본문/힌트)가 단계적으로 보여 정보 위계가 자연스럽다.

## 3) 내보내기(다운로드 모달) 화면
- [ ] 모달 배경, 테두리, 그림자가 패널/카드와 같은 톤으로 맞춰진다.
- [ ] 모달 내부 버튼 변형(primary/secondary/ghost/warn)이 다른 화면과 동일하다.
- [ ] 모달 헤더/바디/푸터 구분선(border)이 선명하지만 거슬리지 않는다.
- [ ] 옵션 그리드 카드들의 라운드/여백이 일정해 스캔이 쉽다.

## 4) 최종 승인 기준
- [ ] 버튼 4단계(primary/secondary/ghost/warn)만으로 대부분 행동 의도가 전달된다.
- [ ] 패널/모달/카드의 그림자와 border 규칙이 통일되어 화면 피로감이 줄었다.
- [ ] 색/간격/반경/폰트 스케일 토큰을 통해 이후 변경 포인트가 예측 가능하다.
