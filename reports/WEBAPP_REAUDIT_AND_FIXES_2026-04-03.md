# 웹앱 재감사 + 핵심 버그 일괄 수정 (2026-04-03)

## A. 저장소 파악 요약 (로컬 전용 기준)

최신 기준 요약: 직접 편집 핵심(리사이즈/추가/복제·삭제/XYWH/z-order/그룹 묶기·해제)과 출력 핵심(JPG/선택 PNG/3x)은 구현 완료다. 현재 미구현 핵심은 선택 모음/컴포넌트 템플릿화와 다중선택 bbox PNG다.

### 1) 현재 편집 흐름
1. 사용자는 `index.html`을 `file://`로 직접 열어 시작한다.
2. HTML/폴더/fixture/붙여넣기 입력 중 하나를 고르면 `src/main.js`에서 `normalizeProject()`를 호출한다.
3. `normalizeProject()`는 script 제거, 자산 경로 해석(상대경로/`uploaded:` 포함), 슬롯 후보 감지, 이슈 목록 생성을 수행한다.
4. iframe `srcdoc`에 정규화 HTML을 넣고 `createFrameEditor()`가 선택/드래그/텍스트/레이어/슬롯 편집을 처리한다.
5. 저장/출력은 브라우저 다운로드 기반으로 진행된다(HTML/PNG/ZIP/리포트).

### 2) 핵심 파일 10개
1. `index.html`
2. `src/main.js`
3. `src/editor/frame-editor.js`
4. `src/core/normalize-project.js`

7. `src/config.js`
8. `src/ui/renderers.js`

### 3) 데스크톱 대비 남은 누락 기능(재감사 기준)
- 선택 모음/컴포넌트 템플릿화
- 내장 HTML 저장/내장→링크 전환 고도화
- 선택 PNG의 다중 선택 bbox 옵션(현재 단일 선택 기준)
- 정밀 단축키 체계(Figma급 밀도)

### 4) 그룹 기능 상태 (구현/제약 분리)
- 구현: 그룹 묶기/해제 버튼 + 단축키(`Ctrl/Cmd+G`, `Shift+Ctrl/Cmd+G`) 동작
- 제약: 잠기지 않은 요소 2개 이상이 필요하고, 공통 조상 자체 선택/이미 그룹 내부 범위에서는 바로 그룹화할 수 없음

### 5) 지금 우선순위 5개
1. 리사이즈 안정화(복수 선택/텍스트 예외)

## D. 남은 위험
- 리사이즈는 현재 단일 선택 중심이며, 복합 레이아웃 부모의 CSS 제약(예: flex/grid)에 따라 체감이 다를 수 있음.
- 추가 요소의 기본 삽입 위치는 `선택 요소 부모` 우선이라 템플릿 구조에 따라 사용자가 수동 재배치 필요.
- 선택 PNG는 현재 “첫 선택 요소 1개” 기준 crop.
- 자동 E2E 회귀는 환경 의존성(`bs4`) 해결 후 재가동 필요.
