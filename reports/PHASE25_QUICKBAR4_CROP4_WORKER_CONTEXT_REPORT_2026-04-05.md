# Phase 25 — Quickbar 3차, Crop HUD 4차, Diff Worker, 섹션 우클릭 빠른 액션, 현실 평가

## 이번 라운드에서 실제로 반영한 것

### 1) 플로팅 퀵바 3차 압축
- 상시 메인 퀵바를 `복제 / 삭제 + 문맥형 2버튼` 구조로 압축했다.
- 이미지 선택 시: `교체 / 크롭`
- 텍스트 선택 시: `텍스트 / 속성`
- 다중선택 시: `정렬 / 간격`
- 나머지 덜 자주 쓰는 동작은 `···` 오버플로 메뉴로 이동했다.
- 결과적으로 메인 캔버스 위에서 한 번에 보이는 액션 수를 의도적으로 줄였다.

### 2) 크롭 HUD 4차
- 크롭 프리셋의 active 상태를 HUD 버튼에서 시각적으로 표시한다.
- iframe 내부 크롭 오버레이에 `안전영역(safe area)` 가이드를 추가했다.
- 이미지 잠금 상태일 때 내부 오버레이에서 더 강한 잠금 상태를 시각화한다.
- 크롭 런타임에 `presetMode`를 유지하고, 팬/휠 줌이 개입하면 `custom`으로 전환한다.

### 3) diff 계산 worker 분리
- 긴 HTML 비교는 메인 스레드 동기 계산 대신 Blob Worker에서 계산하도록 분기했다.
- 기준:
  - base + target combined chars >= 120000
  - 또는 combined lines >= 5200
- worker 사용 시 비교 패널은 `비교 계산 중…` 상태를 먼저 보여주고, 완료 후 chunk를 렌더링한다.
- worker 미지원/실패 환경은 기존 동기 계산으로 fallback 한다.

### 4) 섹션 썸네일 우클릭 빠른 액션
- 카드 우클릭 시 메뉴를 커서 근처에 띄우는 context 모드로 보강했다.
- 메뉴 상단에 `포커스 / 복제 / 삭제` 3개를 빠른 액션으로 배치했다.
- 기존 세부 액션(선택, 범위 선택, 토글, 위/아래 이동 등)은 그대로 유지했다.
- 바깥 영역 클릭 시 열린 섹션 메뉴를 닫도록 정리했다.

## 현실적인 상태 평가

### 목표 대비 구현 수준
이 프로젝트의 핵심 목표는 다음 4개다.
1. **로컬에서 바로 여는 HTML 기반 상세페이지 편집기**
2. **슬롯/이미지/텍스트/섹션을 캔버스에서 편하게 수정**
3. **HTML 코드 편집과 시각 편집을 왕복**
4. **HTML/PNG/JPG로 안정적으로 저장/출력**

현재 수준을 현실적으로 잡으면:
- 기능 완성도: **86% 전후**
- HTML-first 왕복 편집 완성도: **88% 전후**
- 섹션/긴 문서 편집성: **82% 전후**
- 크롭/이미지 손맛: **80% 전후**
- 저장/출력/회귀 안전성: **84% 전후**
- 전체 목표 달성률: **77% 전후**

### 사용성 수준
- 일반적인 "기능은 있는데 못 쓰는 상태"는 확실히 벗어났다.
- HTML 기반 상세페이지 편집기라는 문맥에서는 **실사용 가능한 수준**까지 올라왔다.
- 다만 **미리캔버스급 세련된 질감/몰입감/즉답성**까지는 아직 남았다.

현실적으로:
- 현재 사용성: **70~74점대**
- 미리캔버스급 사용성 달성률: **58~62%**

## 아직 아쉬운 점
1. **전체 컨트롤 수가 아직 많다.**
   - index.html 기준 button 190개, input 24개, select 13개, details 12개
   - 정보구조는 좋아졌지만, 여전히 기능 노출량이 많다.

2. **퀵바가 더 줄어들 여지가 있다.**
   - 지금은 4버튼 구조로 압축됐지만, 문맥별로 더 과감하게 줄일 수 있다.

3. **섹션 썸네일이 저해상도 스크린샷 기반이라 대형 문서에서는 생성 속도/품질 타협이 있다.**

4. **실브라우저 자동 E2E가 없다.**
   - 현재는 file:// 정책 때문에 headless 자동 클릭 스모크가 막힌다.
   - 정적/구조/회귀 검증은 강하지만, 손맛은 여전히 실브라우저 수동 확인이 중요하다.

5. **Miricanvas와의 차이점**
   - Miricanvas는 그래픽 편집기 기준의 초고도 polish가 있다.
   - 우리 앱은 HTML-first라서 강점은 `코드 ↔ 캔버스` 왕복에 있지만, 반대로 UI 미세 폴리시와 시각 통일감은 더 손봐야 한다.
