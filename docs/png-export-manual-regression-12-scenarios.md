# PNG 내보내기 수동 회귀 테스트 (12개 기준 시나리오)

이 문서는 **코딩 초보자도 그대로 따라 할 수 있는** PNG 내보내기 점검표입니다.

- 왜 하나요? → "기본 기능은 되는지", "수정한 내용이 실제 PNG에 반영되는지", "반복 실행해도 안 깨지는지"를 확인하기 위해서입니다.
- 언제 하나요? → 기능 수정 후, 배포 전, 또는 "어제는 됐는데 오늘은 안 됨" 같은 이슈가 생겼을 때 실행합니다.

---

## 0) 시작 전 준비 (공통)

1. 앱 열기: `index.html`을 브라우저에서 엽니다.
2. 테스트용 HTML/이미지 파일을 한 폴더에 준비합니다.
3. 다운로드 폴더를 비웁니다(결과 파일 확인이 쉬워짐).
4. 브라우저 콘솔 에러가 있는지 한 번 확인합니다(F12 → Console).

> 용어 정리
>
> - **회귀(regression)**: 원래 되던 기능이 수정 후 깨지는 현상
> - **E2E 테스트**: 사용자가 실제로 쓰는 흐름(업로드→수정→내보내기)을 끝까지 검증

---

## 1) 기본 성공 경로 (가장 먼저 실행)

**목적**: 최소 기능 정상 여부 확인

### 단계
1. 상세페이지 HTML 업로드
2. 슬롯 이미지 1개 교체
3. 텍스트 1줄 수정
4. `PNG 내보내기` 클릭

### 기대 결과
- `#statusText`에 성공 메시지 표시
- PNG 파일이 실제 다운로드됨
- 파일 크기 0이 아님(권장 기준: 10KB 이상)

---

## 2) 큰 HTML 업로드 안정성

**목적**: 대용량 문서에서 로딩/프리징 확인

### 단계
1. 이미지/섹션이 많은 HTML 업로드
2. 화면 조작(줌/선택/이동) 1~2회
3. PNG 내보내기

### 기대 결과
- 브라우저 프리징 없이 동작
- PNG 다운로드 완료
- 주요 섹션 누락 없음

---

## 3) 이미지 교체 후 즉시 PNG 추출

**목적**: 업로드 직후 반영 누락 버그 탐지

### 단계
1. HTML 업로드
2. `replace image`로 이미지 교체
3. 바로 PNG 내보내기

### 기대 결과
- 바꾼 이미지가 PNG에 반영됨
- 이전 이미지가 보이면 실패

---

## 4) 여러 슬롯 연속 교체

**목적**: 다중 변경 시 최종 상태 반영 확인

### 단계
1. HTML 업로드
2. 슬롯 3개 이상 연속 교체
3. PNG 내보내기

### 기대 결과
- 모든 교체 슬롯이 PNG에 반영
- 일부만 반영되면 실패

---

## 5) 텍스트 수정 + 정렬/이동 후 PNG

**목적**: 레이아웃 변경이 export 결과에 반영되는지 확인

### 단계
1. HTML 업로드
2. 제목/본문 텍스트 수정
3. 위치/정렬 조정
4. PNG 내보내기

### 기대 결과
- 편집 화면과 PNG 배치가 동일
- 텍스트 잘림/겹침 없음

---

## 6) Undo/Redo 이후 PNG

**목적**: 히스토리 꼬임(상태 불일치) 탐지

### 단계
1. 이미지 교체 + 텍스트 수정
2. Undo 2회
3. Redo 2회
4. PNG 내보내기

### 기대 결과
- 최종 편집 상태 그대로 PNG 생성
- Undo 상태가 섞이면 실패

---

## 7) 에러 입력 방어 (잘못된 HTML)

**목적**: 실패 입력에서도 앱 생존성 확인

### 단계
1. 비정상 HTML(태그 누락 등) 입력/업로드 시도
2. 상태 메시지 확인

### 기대 결과
- 앱이 멈추지 않고 오류 안내 문구 출력
- 이후 정상 HTML 재업로드 가능

---

## 8) 상대경로 이미지 포함 HTML

**목적**: 자산 경로 해석 회귀 확인

### 단계
1. 상대경로 이미지 포함 HTML/폴더 업로드
2. unresolved/resolved 상태 확인
3. PNG 내보내기

### 기대 결과
- 가능한 이미지는 정상 로드
- 누락 자산은 안내 문구로 구분
- 내보내기 자체는 실패하지 않음

---

## 9) 연속 PNG 내보내기 (반복 안정성)

**목적**: 1회 성공 후 2~3회차 실패 버그 탐지

### 단계
1. HTML 업로드 후 PNG 1회 내보내기
2. 작은 수정 후 PNG 2회차
3. 다시 수정 후 PNG 3회차

### 기대 결과
- 3회 모두 다운로드 성공
- 파일명 정책(중복 회피/덮어쓰기)이 일관적

---

## 10) 새로고침/재열기 후 동일 결과 재현

**목적**: 임시 상태 의존 버그 탐지

### 단계
1. 편집 후 PNG 내보내기
2. 브라우저 새로고침 또는 앱 재열기
3. 동일 HTML 재업로드 + 같은 편집 반복
4. PNG 내보내기

### 기대 결과
- 1차와 유사한 품질 결과
- 재실행 후에도 기능 정상 동작

---

## 11) 고해상도(또는 3x 프리셋) 출력 점검

**목적**: 해상도 옵션 변경 시 화질/깨짐 확인

### 단계
1. 기본 출력 1회
2. 고해상도(또는 3x) 출력 1회
3. 결과 2개 비교

### 기대 결과
- 고해상도 파일 크기 증가
- 텍스트/이미지 경계가 과도하게 깨지지 않음

---

## 12) 실제 사용자 흐름 종합 E2E

**목적**: 실무 패턴 그대로 최종 검증

### 단계
1. 실제 상세페이지 HTML 업로드
2. 대표 이미지/가격/문구 수정
3. 섹션 순서/간격 조정
4. PNG 내보내기
5. 결과물 공유(메신저/문서 삽입) 테스트

### 기대 결과
- 실사용 가능한 품질 PNG 생성
- 잘림/폰트 깨짐/레이어 누락 없음
