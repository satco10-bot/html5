# 웹앱 1단계 기능 명세서

## 1. 제품 목표

이 프로젝트의 1차 목표는 **기존 HTML 상세페이지를 가져와서, 이미지 슬롯을 자동 인식하고, 이미지를 쉽게 끌어다 넣고, linked HTML / embedded HTML / 전체 PNG / 분할 PNG로 안정적으로 저장하는 로컬-퍼스트 웹앱**의 기준을 고정하는 것입니다.

사용자가 원하는 최종 방향은 단순 HTML 편집기가 아니라, **미리캔버스/피그마처럼 편하게 쓰는 상세페이지 편집 도구**입니다.  
따라서 이번 단계의 목적은 "기능을 많이 붙이는 것"보다 **무엇이 반드시 동작해야 하는지와 어떤 샘플로 합격을 판정할지**를 명확히 정하는 데 있습니다.

## 2. 이번 단계 산출물

- 기능 명세서
- fixture manifest(JSON)
- 기준 샘플 HTML 5종
- acceptance matrix
- fixture sanity check 스크립트

## 3. 북극성 사용자 흐름

1. 사용자가 기존 상세페이지 HTML을 연다.
2. 웹앱이 HTML을 iframe/canvas에 안전하게 로드한다.
3. 이미지 슬롯을 자동 인식한다.
4. 사용자가 이미지를 드래그해서 슬롯에 넣는다.
5. 사용자가 필요하면 슬롯을 수동 지정/해제하고, 크롭/정렬을 조정한다.
6. 사용자가 linked HTML / embedded HTML / 전체 PNG / 분할 PNG로 저장한다.
7. 저장 결과물에서 이미지가 빠지지 않고, 기존 링크/내장 이미지도 유지된다.

## 4. 1차 범위 안의 기능

### 4.1 입력 / 프로젝트 시작
- HTML 파일 열기
- HTML 코드 붙여넣기
- 드래그앤드롭으로 HTML 열기
- 최근 프로젝트 다시 열기
- 샘플 fixture 바로 열기

### 4.2 정규화 / import
- 상대경로 이미지 해석
- `uploaded:` 같은 커스텀 스킴 해석
- `img[src]`, `img[srcset]`, CSS `background-image` 수집
- 문서 내 이미지 라이브러리 생성
- 슬롯 후보 UID 부여

### 4.3 슬롯 감지
- explicit slot(`data-image-slot`, `.image-slot`) 우선 감지
- heuristic slot(`media-shell`, `hero-shot`, `visual`, `ph`, placeholder 문구 등) 감지
- 기존 이미지가 이미 들어간 슬롯도 감지
- 진단 모드에서 감지 이유와 near miss를 표시

### 4.4 편집
- 이미지 1장 드래그로 교체
- 이미지 여러 장 드래그로 순차 배치
- cover / contain / top / center / bottom 프리셋
- 최근 사용 이미지 다시 적용
- 문서 내 이미지 다시 적용
- 슬롯 수동 지정 / 해제
- 전경 슬롯 / 배경 슬롯 전환
- 텍스트 우선 / 박스 우선 / 이미지 우선 선택 모드

### 4.5 저장 / export
- linked HTML 저장
- embedded HTML 저장
- 전체 PNG 저장
- 분할 PNG 저장
- export 전 검수:
  - 빈 슬롯
  - 깨진 자산 경로
  - 기존 이미지 손실 여부
  - 커스텀 스킴 미해결 여부

## 5. 이번 단계에서 아직 하지 않는 것

- 실시간 협업
- 사용자 계정/클라우드 동기화
- AI 자동 배치
- 템플릿 마켓플레이스
- 브라우저 간 완전 동일 UI 보장
- 고급 벡터 도형 편집

## 6. 핵심 데이터 용어

- **Project**: 현재 작업 중인 상세페이지 하나
- **Fixture**: 기능 합격 여부를 판정하는 기준 HTML 샘플
- **Slot**: 이미지를 넣을 수 있는 편집 대상
- **Asset**: 프로젝트에서 사용하는 이미지 파일 또는 내장 이미지
- **Linked HTML**: 이미지를 별도 assets 폴더로 빼는 저장 형태
- **Embedded HTML**: 이미지를 HTML 내부로 포함하는 저장 형태
- **Export Job**: PNG/JPG/HTML 결과물을 만드는 작업 단위

## 7. 품질 기준

### 반드시 만족
- explicit slot fixture는 **오탐 없이 정확한 수를 감지**해야 함
- heuristic fixture는 manifest에 정의한 **최소 필수 슬롯 수**를 감지해야 함
- F05 fixture는 저장/출력 후에도 **기존 `uploaded:` 이미지 2개가 사라지면 안 됨**
- 분할 PNG 순서는 문서의 시각적 섹션 순서와 일치해야 함

### 권장
- 긴 상세페이지에서도 스크롤 편집이 버벅이지 않아야 함
- 최근 이미지 / 문서 이미지 재사용 흐름이 2클릭 안에 끝나야 함
- 빈 슬롯/깨진 링크 경고가 저장 전에 보여야 함

## 8. 기능 우선순위

### P0
- HTML import
- 슬롯 auto-detect
- 드래그 이미지 넣기
- 수동 슬롯 지정/해제
- linked / embedded 저장
- 전체 / 분할 PNG export
- 기존 이미지 보존

### P1
- 진단 오버레이
- 최근 이미지 / 문서 이미지 패널
- 다중 드롭 자동 배치
- 크롭 프리셋
- autosave / 복구

### P2
- 컴포넌트/템플릿 시스템
- CSV/JSON 데이터 주입
- 브랜드 프리셋
- 협업 기능

## 9. 합격 기준 운영 원칙

1. 새로운 기능을 만들기 전에 fixture manifest를 먼저 갱신한다.
2. 고친 뒤에는 최소한 fixture sanity check와 acceptance matrix를 다시 확인한다.
3. 특정 샘플만 맞추는 임시 if문은 금지한다.
4. 버그 수정 후 fixture count가 바뀌면 왜 바뀌었는지 기록한다.
